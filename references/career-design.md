# 迭代式職涯規劃(對應課程 Ch1)

核心隱喻:**把「你」當成一個產品。** 產品不會一次做到完美,而是靠版本迭代逼近。
職涯也一樣:current version → 假設下一版 → 小成本測試 → 反思 → 迭代。

## 兩種互補技巧

### A. 假設測試思維(hypothesis-testing)
別把「我想做 X」當成必須一次賭對的人生決定;把它當成**一個待驗證的假設**。
1. 寫下假設:「如果我轉去做 X,我會更投入 / 更有成長 / 更符合長期目標。」
2. 設計**低成本實驗**去測,而不是直接 all-in:side project、約相關職位的人聊、
   接一個短期任務、上一門課、做一個週末專案。
3. 定義**成功訊號**:看到什麼算假設成立?(做起來有能量、學得快、別人給正回饋)
4. 依結果決定:加碼、轉向、或放棄。每一輪都讓你用更低風險換更多資訊。

### B. 自我反思練習(self-reflection)
方向感不是想出來的,是反思校準出來的。
- 定期(每週 / 每月)回看:**哪些事讓我有能量?哪些事在耗我?**
- 能量訊號比頭銜、薪水更誠實 —— 長期耗能的路再光鮮也走不遠。
- 記錄「做什麼時忘記時間」「做什麼時一直看時鐘」,這是天賦與熱情的線索。

兩者搭配:反思產生假設,測試驗證假設,結果再餵回反思。這就是職涯的迭代迴圈。

## 哈佛商學院職業與生活指南的精神
把「職業」與「生活」一起設計,而不是只優化工作。問的不是「怎麼升最快」,
而是「我要的人生樣貌是什麼,什麼樣的職涯能支撐它」。

## 實戰:設計你的夢想職涯
1. **願景:** 用 3–5 年尺度描述理想的工作 + 生活樣貌(具體,不是「成功」這種空話)。
2. **拆假設:** 達到願景需要哪些前提成立?把它們列成可測試的假設。
3. **排實驗:** 每個假設配一個低成本測試 + 成功訊號 + 時間點。
4. **設反思節奏:** 固定回看能量訊號與實驗結果,迭代下一版。

📤 產出:願景 + 假設清單 + 對應實驗 + 反思節奏。
