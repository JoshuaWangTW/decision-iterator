# 策略思考與商業分析師角色(對應課程 Ch5 + Ch7)

## A. 策略思考(Ch5)

### 策略的首要任務
策略 = **在資源有限下,選擇做什麼、不做什麼**。
洞察告訴你「現況為何」;策略告訴你「往哪走、放棄什麼」。沒有取捨的不是策略,是願望清單。

把洞察轉成策略時,問:
1. 這幾條洞察共同指向哪一個機會 / 威脅?
2. 我們有哪些選項?每個選項要投入什麼、放棄什麼?
3. 哪個選項對北極星指標的槓桿最大、風險最可控?
4. 成功的前提是什麼?哪些假設一旦不成立,策略就垮?

### 策略實戰(把策略落地)
- **連回決策者的目標**:策略要服務 Step 0 定義的決策,不是另開戰場。
- **可執行化**:策略要能拆成行動(誰、做什麼、何時、用什麼指標衡量)。
- **情境化**:給保守 / 樂觀(必要時加悲觀)情境,讓決策者看見區間而非單點。
- **設定領先指標**:除了結果指標(營收),設能早期看出策略是否奏效的領先指標。

## B. 商業分析師的三大核心技能(Ch7)

商業分析師之所以不同於「會跑數據的人」,在於把三件事接起來:

1. **邏輯與問題拆解(Thinking)**
   把模糊商業問題框定成決策、用 issue tree 拆成 MECE 假設。這是上游,決定方向。

2. **數據分析與驗證(Analysis)**
   用數據證實/證偽假設、做拆解與歸因。這是工具層 —— 重要,但只是手段。

3. **策略與溝通(Communication)**
   把發現收斂成策略建議,再用商業簡報讓決策者能行動。這是下游,決定影響力。

三者缺一:只有 1 是空談、只有 2 是技工、只有 3 是包裝。商業分析師要三者貫通,
讓「分析」真正驅動「決策」並「創造價值」。

## C. 轉職 / 切入商業分析(Ch7)

各種職業背景都能轉進商業分析,過往經驗常是加分而非包袱:
- **領域知識(domain)**:原產業的 know-how 讓你更快框定對的問題、看懂數據背後的業務意義。
- **可遷移技能**:邏輯、溝通、簡報、專案管理在哪一行都通用。
- **作品集導向**:用一份完整的 case study(見 `presentation-craft.md` 第 7 節)展示
  你能跑完 0→1 的拆解與決策,比學歷/證照更有說服力。
- **養成路徑**:框架 → 拆解 → 數據驗證 → 策略 → 簡報,逐項補齊;每練一個 case 就累積一件作品。
