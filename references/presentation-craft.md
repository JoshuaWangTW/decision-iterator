# 商業簡報工法(對應課程 Ch6)

簡報不是把分析貼上投影片,而是**為了驅動一個決策而設計的溝通物**。
原則一句話:**先講結論,再講為什麼;一頁一訊息;標題就是你要對方記住的那句話。**

## 目錄
1. Storytelling:簡報的核心
2. 簡報架構與編排
3. 排版邏輯(清晰易懂的關鍵)
4. 精準展示思考脈絡
5. 呈現數據、洞察與策略建議
6. Executive Summary(最關鍵的總結頁)
7. 遞交你的 Case Study

---

## 1. Storytelling:簡報的核心

人記得的是故事,不是數字。把分析串成一條敘事弧:
- **問題(Why now)** → 為什麼這件事現在值得決策者花時間。
- **發現(What we learned)** → 拆解後最重要的 1–3 個洞察。
- **所以該做什麼(So what / Now what)** → 收斂到行動與決策。

每一頁都要能回答:這頁推進了故事的哪一步?推不動的頁,刪掉。

---

## 2. 簡報架構與編排

**金字塔結構(由上而下):**
1. 第 1 頁就是答案 + 建議(BLUF — Bottom Line Up Front)。
2. 接著 3 個支撐論點(對應 3 個洞察)。
3. 每個論點下才放證據(圖表、數據、方法)。

**依受眾分層:**
- 高層 / 決策者:摘要 + 決策 + 風險,證據放附錄。
- 執行層:補方法、口徑、實作細節,讓他們能照著做。

**編排節奏:** 一個論點一個區塊;區塊之間用「過場頁」明確切換主題,別讓聽眾迷路。

---

## 3. 排版邏輯(清晰易懂的關鍵)

排版的目的是**降低理解成本**,不是美化。
- **標題寫結論句**,不是標籤。寫「A 店營收領先主要來自來客數」,不是「營收比較」。
- **一頁一訊息**:一張投影片只證明一件事。資訊太多就拆頁。
- **視覺層次**:最重要的數字 / 結論放最顯眼處;用大小、粗細、顏色引導視線。
- **對齊與留白**:對齊製造秩序感;留白讓重點突出。別塞滿。
- **顏色克制**:一個強調色標重點,其餘灰階。顏色一多就沒有重點。

---

## 4. 精準展示思考脈絡

決策者要信你的結論,得看得到你怎麼想到的。
- 把假設樹 / 拆解邏輯**視覺化**(一張圖說明你怎麼拆問題)。
- 展示「我們驗了什麼、證實/證偽了什麼」—— 包含被推翻的假設(增加可信度)。
- 讓推理鏈可追溯:從數據 → 洞察 → 建議,每一跳都看得到根據。

---

## 5. 呈現數據、洞察與策略建議

**圖表選擇對齊訊息:**
- 趨勢/隨時間 → 折線圖;組成/佔比 → 堆疊長條 / 100% 堆疊;比較 → 長條;
  關係 → 散點;漏斗 → 漏斗圖。
- **每張圖標題寫它的 so-what**,不是寫它是什麼圖。
- 圖上直接標註要看的那個點(箭頭、圈、註解),別讓聽眾自己找。

**從數據到建議的呈現順序:**
數據(發生什麼)→ 洞察(為什麼重要、量級)→ 策略建議(所以做什麼)。
三者要在視覺上分得清,別把「發現」和「建議」混成一團。

---

## 6. Executive Summary(最關鍵的總結頁)

這頁通常是唯一被高層讀完的頁。獨立成立、不靠口頭補充:
- **結論:** 一句話答案 + 建議。
- **三個支撐:** 各一句,對應三個洞察(帶量級)。
- **下一步:** 誰、做什麼、何時。
- (選配)**風險 / 前提:** 這個建議成立的關鍵假設。

寫法:即使對方只看這一頁,也能做出正確決策。

---

## 7. 遞交你的 Case Study

把整份分析打包成可遞交的 case study(求職作品集 / 對內提案皆適用):
- **結構:** 背景 → 問題框定 → 假設與拆解 → 分析與洞察 → 策略建議 → 結論/下一步。
- **展示思路勝過展示結果:** 評審/主管想看的是你的拆解邏輯與判斷,不只是漂亮結論。
- **誠實標註限制:** 哪些是已驗證、哪些待補、用了什麼假設。誠實提升專業可信度。
- **收尾回到價值:** 這個分析替決策者省下/賺到/避免了什麼。
