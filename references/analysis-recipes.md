# 拆解實作 Recipes(對應課程 Ch4)

當 Step 3 要把一個指標拆開驗證時,套用下面對應的 recipe。
每個 recipe 的精神都一樣:**先寫出恆等式(把指標拆成乘法/加法因子),再決定哪個因子在動。**
別急著切維度;先有結構,切維度才有意義。

## 目錄
1. 前置作業(資料就位)
2. 訂單總數拆解
3. 分店營運表現(亮眼 vs 落後)
4. 購買頻率拆解
5. 平均客單價拆解
6. 通用陷阱

---

## 1. 前置作業(資料就位)

在拆任何東西前:
- 把 Raw Data 放進工作表(Google Sheets / Excel),欄位命名一致、日期格式統一。
- 對齊比較期間(同樣週數、剔除蜜月期、連假、天災等異常日)。
- 確認折扣 / 退款 / 取消的口徑一致 —— 否則營收、客單價全部會被污染。
- 先做一輪資料健檢:缺值率、極端值、重複單、時間斷層。

口徑沒釘死之前,後面所有數字都不可信。

---

## 2. 訂單總數拆解

**恆等式:** 訂單總數 = 來客數(交易筆數) ≈ 流量 × 轉換率
或從客群面:訂單數 = 新客訂單 + 既有客訂單(回購)

**拆解步驟**
1. 先看訂單總數的趨勢線,標出轉折點(哪一週/哪一天開始變)。
2. 拆「新客 vs 回購」:是拉新出問題,還是留存出問題?
3. 切維度:時段、星期、通路、品類、地區 —— 找出貢獻變化最大的切片。
4. 把總變化量歸因到各切片(誰貢獻了 +X%、誰 −Y%),確認加總對得起來。

**so-what 方向:** 訂單數是「來的人多不多」的代理。若落差主要在訂單數而非客單價,
問題通常在「立地 / 行銷 / 留存」,而不是「定價 / 組合」。

---

## 3. 分店營運表現(亮眼 vs 落後)

這是多店比較的核心 recipe。比較兩家(或多家)店時:

**第一刀永遠是:** 營收落差 = 來客數貢獻 + 客單價貢獻。
先算出落差有幾成來自「人來得多」、幾成來自「買得貴」。這一刀通常砍掉一半假設。

**接著:**
1. 亮眼店:它贏在哪個因子?(來客 / 連帶率 / 高價品佔比 / 時段覆蓋)
2. 落後店:它輸在哪個因子?是同一個因子的反面,還是不同問題?
3. 分離「可複製 vs 不可複製」:贏的因子裡,哪些是經營能複製(品項組合、陳列、
   時段人力),哪些是地利天賦(特定商圈、外國人客群)不可複製。
4. 用「可複製因子」推導行動;用「不可複製因子」修正對其他店的營收期望。

**so-what 方向:** 比較分析的價值不是「排名」,而是「把贏家的可複製要素變成 playbook」。

---

## 4. 購買頻率拆解

**恆等式:** 營收 = 顧客數 × 購買頻率 × 平均客單價
購買頻率 = 期間內總訂單數 / 活躍顧客數

**拆解步驟**
1. 算出每位顧客的購買次數分佈(不要只看平均 —— 平均會被重度買家拉高)。
2. 用 cohort 看:不同月份進來的顧客,後續回購曲線長怎樣?
3. 分群:高頻 / 中頻 / 一次性顧客各佔多少營收?(常是少數高頻撐大盤)
4. 找「從一次性 → 回頭」的轉換點:第二次購買通常是留存的關鍵門檻。

**so-what 方向:** 頻率低 → 留存 / CRM / 會員機制問題;
頻率集中在少數人 → 大盤對重度顧客有集中風險。

---

## 5. 平均客單價拆解

**恆等式:** 客單價 = 營收 / 訂單數 = 連帶率 × 平均品項單價
連帶率 = 品項數 / 訂單數(一張單買幾樣)

**拆解步驟**
1. 客單價變化,是「買更多樣」(連帶率)還是「買更貴的」(品項單價 / 組合)?
2. 看品類佔比:高價品(禮盒、套組)佔比變化如何牽動客單價?
3. 切折扣影響:折扣口徑、促銷檔期是否在稀釋客單價?
4. 分客群:不同客群的客單價結構差異(觀光客 vs 在地常客)。

**so-what 方向:** 要拉客單價,先確認槓桿在「連帶率」還是「品項組合」——
兩者的行動完全不同(前者靠搭售/陳列,後者靠定價/選品)。

---

## 6. 通用陷阱

- **只看平均、不看分佈** —— 平均藏住兩極;務必看分佈與分群。
- **辛普森悖論** —— 大盤上升但每個切片都在跌(或反之);切維度才看得出。
- **比較期間不對齊** —— 蜜月期、連假、季節性沒剔除,結論失真。
- **歸因加不回原數** —— 各切片貢獻量加總要對得起總變化,否則漏了因子。
- **拆到一半就下結論** —— 第一刀(量 vs 價)只是開始,不是答案。
