# 頂尖 PM 的問題拆解流程(對應課程 Ch2)

精神:**先想清楚再動手。** 拆解的順序是 why → 拆成子問題 → 發想 → 收斂 → 決策。

## 1. 從「為什麼」開始(目標與優先順序)
動手解題前先釘住:
- 這題真正要達成的目標是什麼?(常和表面問題不同)
- 在所有能做的事裡,為什麼是這題優先?
搞錯 why,後面拆得再漂亮都是用力在錯方向。

## 2. 樹狀思考 +「我們如何能?」(HMW,Meta)
- 把大問題用**樹狀結構**拆成 MECE 子問題(彼此互斥、合計窮盡)。
- 把每個子問題改寫成「**我們如何能…?(How Might We)**」的形式 ——
  這個語法把「問題」轉成「可發想的機會」,打開解法空間而不是卡在抱怨。
- 例:問題「用戶流失」→ 子問題「新手沒上手」→ HMW「我們如何能讓新用戶第一天就成功一次?」

## 3. 發想解法(SCAMPER + 瘋狂 8)
先求量再求質,別太早收斂:
- **SCAMPER:** 用七個動詞逼出變化 —— Substitute(替換)、Combine(結合)、
  Adapt(調整)、Modify(修改/放大)、Put to other use(他用)、Eliminate(刪除)、
  Reverse(重排/反向)。對著現況逐一套,逼出沒想過的點子。
- **瘋狂 8(Crazy 8s):** 8 分鐘逼自己畫 / 寫 8 個點子。時間壓力打破完美主義,逼出數量。

## 4. 加快決策的思考框架
- **衝擊評估(impact assessment):** 用「影響力 × 可行性 / 成本」排序選項,
  把資源押在高影響、可執行的那幾個,而不是平均分散。
- **陽光測試(sunshine test):** 「如果這個決定被攤在陽光下、公開檢視,我會不會心虛?」
  ——用來快速過濾道德 / 長期風險上站不住腳的選項。

## 5. 哈佛 MBA 分析潛在商業機會
評估一個機會(內部專案或新事業)時,結構化地看:
- 問題夠不夠痛、市場 / 受眾夠不夠大、我們有沒有獨特優勢、可行性與風險、
  以及「不做會怎樣」。把機會當假設來驗,而非憑感覺投入。

📤 產出:why 陳述 → HMW 子問題樹 → 發想清單 → 依衝擊排序的選項 → 決策(過陽光測試)。
